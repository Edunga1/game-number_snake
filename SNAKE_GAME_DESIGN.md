# 스네이크 + 수박게임 + 2048 퓨전 게임 기획서

## 컨셉 요약

스네이크를 조작해 숫자 먹이를 먹고, 같은 숫자끼리 합쳐서 점수를 올리는 퍼즐 아케이드 게임.
스네이크 조작 + 수박게임 합체 쾌감 + 2048 숫자 전략이 결합된 구조.

---

## 핵심 규칙

### 이동 & 먹기
- 플레이어는 스네이크를 조작하여 필드 위의 숫자 먹이를 먹는다.
- 뱀의 **머리가 가장 큰 숫자**이다.
- 머리 숫자 **이하**의 먹이만 먹을 수 있다.
- 머리보다 **큰 숫자**를 먹으면 죽거나 패널티.

### 꼬리 & 합체
- 먹은 먹이는 **꼬리 끝에 붙는다**.
- 꼬리에서 **인접한 같은 숫자**끼리 합쳐진다.
- 합쳐진 결과가 또 그 앞 숫자와 같으면 **연쇄 합체** 발생.
- 연쇄가 이어지면 머리까지 합쳐질 수 있다. (핵심 쾌감 포인트)

### 라운드 클리어
- 라운드마다 **요구 점수**가 있다.
- 점수를 채우면 화면 외곽의 **전기장이 해제**된다.
- 전기장이 사라지면 뱀이 방을 나가서 다음 라운드로 진행.

## 우려점 & 개선 방안

### 1. 뱀 길이 vs 조작 난이도 밸런스
- **문제:** 합체하면 짧아져서 스네이크 긴장감이 사라짐. 안 합쳐지면 너무 어려움.
- **개선안:**
  - 합체 시 길이는 줄지만 몸통 크기(두께)가 커져서 공간 압박 유지
  - 라운드가 올라갈수록 맵이 좁아지거나 장애물 추가
  - 일정 길이 이하로 줄면 보너스 먹이가 쏟아져서 다시 길어지는 사이클

### 2. 숫자 배치 랜덤성 (억울한 죽음)
- **문제:** 머리보다 큰 숫자가 길을 막으면 회피 불가.
- **개선안:**
  - 큰 숫자 스폰 전 경고 표시 (바닥 그림자)
  - 즉사 대신 체력 시스템 (3회 기회)
  - 큰 숫자를 밀어내는 스킬 (쿨타임 있음)
  - 먹을 수 없는 숫자는 색상으로 명확히 구분 (빨간색 = 위험)

### 3. 화면 복잡도
- **문제:** 뱀 숫자 + 필드 숫자 + 전기장 = 시각 과부하.
- **개선안:**
  - 숫자 대신 크기로 주로 표현 (수박게임처럼), 숫자는 보조 표시
  - 뱀 몸통은 단색 그라데이션, 필드 먹이는 다른 색상 계열로 분리
  - 전기장은 화면 테두리 이펙트로 간결하게 처리

### 4. 연쇄 합체 빈도
- **문제:** 핵심 쾌감인 연쇄가 너무 드물면 재미 없음.
- **개선안:** 프로토타입 단계에서 숫자 스폰 확률을 조절하며 밸런스 테스트 필요.

### 5. 합체 타이밍 — 자동 합체 + 유예 연출
- **문제:** 합체가 즉시 처리되면 뭐가 일어났는지 인지하기 어렵고, 수동 합체는 조작이 복잡해져 스네이크의 빠른 템포를 해침.
- **개선안:**
  - **자동 합체** 채택 (별도 조작 불필요)
  - 합체 시 **0.3~0.5초 유예 연출** 적용 — 몸통이 빛나거나 떨리는 이펙트 후 합체
  - 연쇄 합체 시 톡톡톡 순차적으로 합쳐지는 연출 → 수박게임 특유의 쾌감 재현
  - 유예 시간 동안 "다음에도 합쳐질까?" 하는 기대감 유발

### 6. 머리 숫자 성장 밸런스 — 감소 메카닉
- **문제:** 머리가 커지면 모든 먹이를 먹을 수 있어 난이도가 급락함. 라운드별 숫자 범위를 올리는 것만으론 부족.
- **개선안:**
  - **머리 숫자 유지 비용** — 일정 거리 이동 시마다 머리 숫자 1 감소
  - 큰 머리를 유지하려면 계속 합체해야 하는 긴장감 유지
  - "머리가 커졌으니 빨리 큰 먹이를 활용해야 한다"는 자연스러운 타임 프레셔
  - 머리가 0이 되면 사망 → 적극적 플레이 유도
  - (대안) 너무 복잡하면 라운드마다 머리 숫자 리셋으로 단순화 가능

### 7. 꼬리 순서 전략 — UI로 부각
- **문제:** 먹는 순서에 따른 꼬리 배열이 핵심 전략이지만, 플레이어가 인지하기 어려움.
- **개선안:**
  - **꼬리 미리보기 UI** — 화면 한쪽에 현재 꼬리 숫자 배열을 일렬로 표시
  - **합체 가능 하이라이트** — 필드에서 현재 먹으면 합체가 발생하는 먹이를 살짝 반짝이게 표시
  - 초보자도 "저걸 먹으면 합쳐지겠구나"를 직관적으로 인지 가능
  - 튜토리얼에서 "같은 숫자를 연속으로 먹어봐" → 연쇄 합체 체험 → 자연스러운 학습

## 프로토타입 우선 검증 포인트

| 순위 | 검증 항목 | 이유 |
|------|-----------|------|
| 1 | 합체 유예 연출 | 쾌감의 핵심. 연출 없으면 수박게임 감성이 안 살아남 |
| 2 | 머리 숫자 감소 메카닉 | 밸런스의 핵심. 이게 있고 없고에 따라 게임 난이도 곡선이 완전히 달라짐 |
| 3 | 꼬리 미리보기 UI | 전략성의 핵심. 없으면 운 게임처럼 느껴질 수 있음 |

## 개발 메모
- MVP는 라운드 모드만 구현
- **플랫폼: 웹(HTML Canvas + TypeScript) 추천** — 설치 없이 브라우저에서 즉시 테스트, URL 공유로 피드백 수집 용이. 재미 검증 후 Unity/Godot 등으로 이식 고려
- 프로토타입을 빠르게 만들어서 합체 밸런스를 실제 플레이로 검증하는 것이 우선
